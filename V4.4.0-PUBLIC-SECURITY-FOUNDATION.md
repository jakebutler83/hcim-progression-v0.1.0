# v4.4.0 Public Security Foundation

This release moves group creation and invite acceptance out of browser-side Firestore writes and into authenticated Netlify Functions.

## Added

- `netlify/functions/group-create.js`
- `netlify/functions/group-invite-accept.js`
- Hardened `firestore.rules`
- Server-side validation for group names, group size, membership capacity and member slots
- Hashed invite document IDs for newly created groups
- Compatibility with existing legacy invite documents so current groups keep working
- Protection against client-side edits to `ownerUid`, `memberUids`, `memberCount`, roles and slots

## Required deployment steps

1. Deploy this whole website folder to Netlify.
2. Confirm the Firebase Admin environment variables are still configured in Netlify.
3. In Firebase Console, open Firestore Database > Rules.
4. Replace the existing rules with the contents of `firestore.rules`.
5. Publish the rules only after the Netlify deployment completes.
6. Test with the current owner account, then test joining with a separate account.

## Existing group compatibility

Existing group invite codes remain accepted. New groups use hashed invite document IDs. The visible invite code remains available to group members in this foundation release; expiring/revocable invite management is planned for the next security milestone.
